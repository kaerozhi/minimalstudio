家里有一台 NAS，装机多年，主系统是 Unraid，目前已经升级至 7.1.4。作为一个菜鸟，也就是最基础的那些应用，日常最高频就是 Jellyfin 追番刷剧看电影，Immich 备份管理照片，Syncthing 备份重要文档，别的功能都算添头和玩具。这么多年运行稳定，最近突发奇想，想配置一个 OpenWRT 作为旁路由，但主板是单网口，无法直通，未免美中不足，所以就在闲鱼上收了一块 Intel I226-V 网卡作为补充。

网卡上机之后，系统可以正常识别。

![系统设备中可以看到三个网口](https://media.kaerozhi.com/2025/07/9c54c04fd6fb6776925921c9549f6483.webp)

进入「设置」中的「网络设置」，默认是这样的：

![默认](https://media.kaerozhi.com/2025/07/159791881366b1dfd18417eb222c3621.webp)

`192.168.7.9` 是旁路由的 IP，但奇怪的是，如果就这样把绑定的 eth1、eth2 解绑，马上就会丢 IP，连不上 Unraid 的管理界面。即便试着把网线插到另外两个网口，也都一样，真是头大。

当然，此时也不必慌张，把 Unraid 的管理 U 盘插到电脑上，用记事本或者别的编辑软件打开 `config/network.cfg`，找到：

```
BRNICS[0]="eth0"
```

将其改为：

```
BRNICS[0]="eth0, eth1, eth2"
```

然后重启 Unraid，片刻后刷新就能找回 Unraid 管理界面了。当然，接屏幕键盘鼠标也可以搞定，但个人觉得不如拔插 U 盘省事。

仔细观察，三个网卡的 MAC 地址分别：

| 接口 | MAC 地址 |
| ---  | --- |
| eth0 | 02:76:C6:00:9D:86 |
| eth1 | 02:76:C6:00:9D:87 |
| eth2 | 3C:7C:3F:7D:EF:59 |

很明显，问题就出在接口的识别上，这两个相连的 86、87 是插上去的 PCI 网卡，那个 59 才是板载的网卡，那如何调整它们的顺序，把 eth0 设置为正确的接口呢？
